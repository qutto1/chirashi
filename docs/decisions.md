# 意思決定ログ

## 2026-07-15: Webアプリ「LINEへ送信」の中継方式

**背景**: Webアプリ（公開GitHub Pages）の「LINEへ送信」を、毎朝の通知先（ボット→登録2ユーザーへのプッシュ）と同じ宛先へ直接送るには、LINEチャネルトークンを使うプッシュ送信が必要。トークンは公開ページに置けないため中継サーバーが要る。

**提示した選択肢**:
1. **GAS中継（推奨・採用）** — Google Apps Scriptを無料の中継サーバーにする。ユーザーのGoogleアカウントで常時稼働、トークンはScript Propertiesに安全保管。初回のみコードを貼り付けてデプロイし、URLを設定。PCの電源状態に依存しない。
2. PC+トンネル中継 — PCで小さな中継サーバー＋常設Cloudflareトンネル。既存トークンを流用できるがPC起動・トンネル維持が前提。
3. 共有ピッカーのまま — 現状の `line.me/R/share`（送信時に宛先を選ぶ）を維持。中継サーバー不要。

**選択**: 1. GAS中継

**実装方針**:
- GASスクリプト（`gas/line-relay.gs`）: doPostで `{secret, text}` を受け、Script Properties のトークン・user_ids でLINE multicastプッシュ。共有シークレットで簡易保護。宛先はサーバー側固定なので、万一エンドポイントが漏れても送信先は本人のLINEのみ。
- Webアプリ: 設定画面にGAS URL・シークレットを入力（localStorageに端末ローカル保存、公開ソースには含めない）。未設定時は従来の共有ピッカーにフォールバック。
